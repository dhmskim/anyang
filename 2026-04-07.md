### 2.3 rag.js Ollama URL 환경변수로 변경

- **변경**: `const OLLAMA_URL = process.env.OLLAMA_URL || 'http://localhost:11434'`
- `dotenv` require 추가

### 2.4 todayStr() 타임존 수정
- **이전**: `new Date().toISOString().slice(0, 10)` → UTC 기준 (KST 자정~09시 날짜 불일치)
- **변경**: KST(+9시간) 보정 후 날짜 추출
```javascript
function todayStr() {
    const now = new Date();
    const kst = new Date(now.getTime() + 9 * 60 * 60 * 1000);
    return kst.toISOString().slice(0, 10);
}
```

## 3. 챗봇 의도 감지 추가

예약 플로우 진행 중에도 사용자 의도를 우선 감지하도록 개선.

### 문제
- 이름 입력 단계에서 "주차예약"이라고 입력하면 이름으로 저장됨
- 차량번호 단계에서 "취소해줘" 입력하면 차량번호 검증 실패

### 해결
- `detectIntent()` 함수 추가: 키워드(예약, 취소, 혼잡, 상담원 등) 포함 시 플로우 리셋 후 해당 기능 실행
- 한글 2~3자 입력(김민석, 홍길동 등)은 이름으로 판단 → 의도 감지 건너뜀
- `inputName()` 검증 강화: 한글/영문만 허용, 2~10글자 제한

### 의도 감지 키워드
| 키워드 | 동작 |
|--------|------|
| 혼잡, 현황, 잔여, 남은자리 | 혼잡 현황 표시 |
| 예약, 주차예약, 예약하고 | 예약 플로우 재시작 |
| 취소, 변경, 취소하고 | 취소 플로우 시작 |
| 상담원, 상담, 연결 | 상담원 연결 |

## 4. 시스템 프롬프트 전면 개편

기존 29줄 → **151줄**으로 대폭 확장.

### 추가된 내용
- **5단계 예약 수집 흐름**: 이름 → 차량번호 → 날짜 → 시간대 → 최종 확인
- **차량번호 4가지 패턴 정규화**: 신형 일반, 지역명 신형, 구형 지역포함, 구형 숫자시작
- **이름 검증**: 한글 2~5자, 업무 단어(예약/주차/조회) 감지 시 의도 감지 모드
- **날짜 인식**: "내일", "다음주 월요일", "6월 3일" 등 자연어 + 과거/60일 초과 거부
- **시간대 인식**: 12/24시간제 혼용, 운영시간(08~22시) 외 거부
- **의도 감지**: 플로우 중 다른 의도 입력 시 맥락 유지하며 안내 후 복귀 유도
- **말투 규칙**: 친절한 존댓말, 명령형/기계적 표현 금지 (구체적 예시 포함)
- **3회 실패 시** 관리사무소 안내
- **반복 정책 질문 규칙**: 일반 규칙 먼저 안내 후 다가오는 날짜 보충
  - 예) "매주 월요일 휴원이에요. 다가오는 휴원일은 4월 14일(월)입니다."

## 5. 사용자/관리자 포트 분리

### 이전
- 사용자 + 관리자 모두 `localhost:3000` (경로로 구분)

### 변경
- **사용자 포털**: `localhost:3000` (parking-portal/ 정적 파일)
- **관리자 페이지**: `localhost:3001` (별도 Express 앱)
- API 라우트는 메인 앱(3000)에서 관리자 앱(3001)으로 위임
- `.env.example`에 `ADMIN_PORT=3001` 추가
- `start.bat`: 서버 시작 → 4초 대기 → 브라우저 2개 열기

### 버그 수정
- `express.static(파일경로)` → `res.sendFile()` 변경 (static은 디렉토리만 지원)
- `path.join(__dirname, '..')` → `path.resolve(__dirname, '..')` (sendFile 절대경로 필요)
- `start.bat`: 브라우저 먼저 열림 → 서버 시작 후 4초 대기 후 열기로 변경

## 6. 관리자 로그인/회원가입/승인 시스템

### 6.1 관리자 로그인 모달
- `prompt()` 방식 제거 → 전체 화면 로그인 오버레이 (CSS 모달)
- 다크 배경 + 화이트 카드 디자인
- 아이디/비밀번호 입력 → JWT 토큰 발급 → localStorage 저장
- 로그아웃 시 토큰 삭제 → 로그인 모달 재표시
- 토큰 유효 시 자동 로그인 (새로고침 시에도 유지)

### 6.2 관리자 회원가입
- 로그인 모달에서 "회원가입" 링크로 전환
- 아이디(영문숫자 4~20자), 비밀번호(8자+), 비밀번호 확인, 이름, 연락처 입력
- `POST /api/auth/admin-signup` → `role: 'admin'`, `status: 'pending'`으로 저장
- "가입 신청이 완료되었습니다. 최고관리자 승인 후 로그인 가능합니다." 표시
- 승인 전 로그인 시도 → "승인 대기 중인 계정입니다" 거부

### 6.3 최고관리자 승인/거절
- 최고관리자 (`ADMIN_ID`, 기본 `AD`)만 승인/거절 가능
- 유저 관리 테이블에서 `pending` 상태 유저에 승인(체크)/거절(금지) 버튼 표시
- `POST /api/admin/users/:id/approve` → `status: 'active'`로 변경
- `POST /api/admin/users/:id/reject` → 유저 데이터 삭제
- `superAdminOnly` 미들웨어 추가 (최고관리자 ID 검증)

### 6.4 서버 API 추가
| Method | URL | 인증 | 설명 |
|--------|-----|------|------|
| `POST` | `/api/auth/admin-signup` | - | 관리자 가입 신청 (pending) |
| `POST` | `/api/admin/users/:id/approve` | JWT (최고관리자) | 가입 승인 |
| `POST` | `/api/admin/users/:id/reject` | JWT (최고관리자) | 가입 거절 (삭제) |

### 6.5 CSS 추가
- `.admin-auth-overlay`: 전체 화면 다크 배경 (`#1a1a2e`, z-index: 99999)
- `.admin-auth-box`: 420px 카드, 그라데이션 헤더
- `.admin-auth-field`: 입력 필드 (포커스 시 초록 보더)
- `.status-dot.pending::before`: 주황색 상태 점
- `.approve-btn` / `.reject-btn`: 승인/거절 버튼
